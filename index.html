For anyone curious about setting up a permanent alias, I was able to configure it appropriately. To permanently create an alias, we need to edit the shell configuration file. First, using "echo $SHELL" , we can see which shell we are using. In this case, its zsh. Now, we need to open the zsh shell configuration file. We can do so by utilizing the command "nano ~/.zshrc" which will open the shell configuration file in the nano text editor application. After some navigation, we'll be prompted with a section that displays some of the default aliases that are pre-programmed into the user shell. we can technically add the alias anywhere (obviously not in the middle of some code; this can be destructive), as long as its on a separate line. I felt comfortable appending my alias where I could see the default aliases listed, that way I avoid accidentally damaging some code. In this case, we would add "alias mypasswdfile ='cat /etc/passwd' . As mentioned previously, we want "cat /etc/passwd" to function as "mypasswdfile'", thus we would type "alias mypasswdfile='cat /etc/passwd'. We can then save and exit the nano editor by pressing ctrl+o in nano, then ctrl+x to exit. Now, we need to reload the configuration file to apply the changes we made. We can do that by inputting "source ~/.zshrc" . Lastly, we can close the terminal and reopen it and verify the alias. To verify it, we can input "alias mypasswdfile" which should then output "alias mypasswdfile='cat /etc/passwd' ". Now, typing our newly created alias "mypasswdfile" will display the /etc/passwd contents. We will also see this command listed when we type alias into our terminal alongside some of the default alias commands. While this was not necessary to complete question 5, I found troubleshooting and learning commands, as well as the navigation of the nano text editor to be extremely rewarding and helpful. I hope this helps anyone who was curious.
